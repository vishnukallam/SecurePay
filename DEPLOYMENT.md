# Production Cloud Deployment Walkthrough

This guide details the step-by-step instructions to deploy the secure UPI Payment application to production using **Neon** (PostgreSQL database), **Render** (Node/Express backend), and **Vercel** (React/Vite frontend).

---

## 1. Neon Database Setup

[Neon](https://neon.tech/) provides a fully managed serverless PostgreSQL database.

### Steps:
1. Sign up/Log in to **[Neon Console](https://console.neon.tech/)**.
2. Click **Create New Project**.
   * **Project Name**: `securepay-db`
   * **PostgreSQL Version**: `16` (or latest stable)
   * **Region**: Choose the region closest to your target users (or close to Render's region, e.g., US East / Oregon).
3. Once the database is provisioned, you will see a connection screen.
4. Copy the **ConnectionString** (PostgreSQL URI). It will look similar to this:
   ```text
   postgres://<username>:<password>@<neon-host-id>.neon.tech/neondb?sslmode=require
   ```
   > [!IMPORTANT]
   > Keep this URI completely private. It contains your master username and database password.

*Note: The SecurePay Node.js backend features automatic table creation and schema provisioning. You do **not** need to manually run any SQL table creation scripts; once the backend successfully connects to Neon, it will create all ledger tables automatically on startup.*

---

## 2. Render Backend Deployment

[Render](https://render.com/) is used to host the Express API server securely.

### Steps:
1. Sign up/Log in to **[Render Dashboard](https://dashboard.render.com/)**.
2. Click **New +** and select **Web Service**.
3. Connect your Git repository (GitHub/GitLab) or input the public repo URL.
4. Set the following Service Parameters:
   * **Name**: `securepay-api`
   * **Language**: `Node`
   * **Branch**: `main`
   * **Root Directory**: `backend` *(Critical: This tells Render to compile and deploy inside the backend subdirectory)*
   * **Build Command**: `npm install && npm run build`
   * **Start Command**: `npm start`
   * **Instance Type**: `Free` (or any tier of your choice)

5. Click **Advanced** to add the following **Environment Variables**:

| Variable | Value / Example | Description |
| :--- | :--- | :--- |
| `NODE_ENV` | `production` | Enables Express security optimization flags. |
| `PORT` | `10000` | Render assigns this automatically, but setting it explicitly is safe. |
| `DATABASE_URL` | *`your_neon_postgresql_uri`* | Connection string copied from Neon in Step 1. |
| `JWT_SECRET` | *`a_long_secure_random_string`* | Cryptographic salt used to sign authentication JWT tokens. |
| `PIN_ENCRYPTION_KEY` | *`64_hex_character_key_32_bytes`* | Hexadecimal encryption key used for AES-256-GCM transaction PIN security. |
| `FRONTEND_URL` | *`https://your-securepay-app.vercel.app`* | URL of the frontend (get this in Step 3, then update here to satisfy CORS). |

6. Click **Deploy Web Service**. Render will install dependencies, compile the TypeScript files to `/dist`, start the Express server, and automatically initialize the Neon tables.

---

## 3. Vercel Frontend Deployment

[Vercel](https://vercel.com/) is used to deploy the React + Vite static web interface with optimized CDN caching.

### Steps:
1. Sign up/Log in to **[Vercel Dashboard](https://vercel.com/)**.
2. Click **Add New** and select **Project**.
3. Select your Git repository.
4. Configure the Project Settings:
   * **Framework Preset**: `Vite`
   * **Root Directory**: `frontend` *(Critical: Tells Vercel to work inside the React frontend folder)*
5. Expand the **Build and Development Settings**:
   * **Build Command**: `npm run build` (or `vite build`)
   * **Output Directory**: `dist`
6. Expand **Environment Variables** and add the API connection variable:

| Variable | Value / Example | Description |
| :--- | :--- | :--- |
| `VITE_API_URL` | *`https://securepay-api.onrender.com`* | The live HTTPS URL generated by Render in Step 2. |

7. Click **Deploy**. Vercel will bundle the M3 styled dashboard assets, wire up the routes, and provide you with a production URL (e.g., `https://securepay-app.vercel.app`).

### Why `vercel.json` is Important:
We have pre-configured a `vercel.json` rewrite file in your frontend root directory. This ensures that when page refreshes occur on protected dashboard subroutes (like `/wallet` or `/transfer`), Vercel serves the main React bundle (`index.html`) rather than triggering a `404 Not Found` error:
```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

---

## 4. Final Security Check & Cross-Origin Handshake

Once both platforms are successfully active:
1. Copy the production URL of your Vercel frontend (e.g., `https://securepay-example.vercel.app`).
2. Go back to your **Render Dashboard** → **securepay-api Web Service** → **Environment Variables**.
3. Update the `FRONTEND_URL` environment variable value to match your actual Vercel URL.
4. Save the changes. Render will automatically redeploy the backend with the new allowed CORS rules, allowing secure secure payouts and ledger transfers to handshake correctly!
